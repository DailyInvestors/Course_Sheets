Web Application Penetration Testing Payloads Guide
This guide provides a structured collection of common payloads used in web application penetration testing. These payloads are designed to identify various vulnerabilities, primarily Cross-Site Scripting (XSS), SQL Injection, and Directory Traversal. Each entry includes a description of the payload's purpose, the vulnerability type it targets, and a list of example code snippets.

Note: This is a basic educational guide. In future notes, each category will be further seperated and more payloads added by category.

Payloads

1. Basic Script Tag Injection
Description: The most straightforward Cross-Site Scripting (XSS) payloads, attempting to execute JavaScript directly within <script> tags. These are often the first payloads to test when initial filtering is minimal. They aim to break out of existing HTML contexts and insert a new script block.
Type: HTML Script Tag Injection (XSS)

script>alert('XSS');</script>
